# 优化目标与验收标准

## 一、核心诉求与优先级

| 优先级 | 目标 | 说明 |
|--------|------|------|
| P0 | 合规 | 密钥管理、操作审计独立模块；敏感操作经 core/compliance 收口 |
| P1 | 稳定性 | 异步任务 + 断点续传 + 失败重试，耗时操作非阻塞 |
| P2 | 可扩展性 | 新增平台仅需新增 adapter 与可选 platform_config，不改 service |
| P3 | 性能 | 数据看板缓存、并行拉取、转码异步化 |

## 二、验收标准

- **解耦**：各平台上传/拉取逻辑解耦，新增平台适配成本 < 1 人天（仅新增 adapter 与 platform_config）。
- **合规**：所有敏感操作（账号读写、上传、数据拉取）经 core/compliance 校验与审计日志记录。
- **双路径**：结构支持「官方 API」与「模拟操作」两种实现，可配置切换（如小红书）。
- **可维护性**：各平台参数独立于 `config/platform_config`，修改平台参数不修改业务代码。

## 三、关键原则（贯穿全流程）

- **合规优先**：密钥管理、操作审计独立模块，不与业务代码混写；合规能力统一收口。
- **双路径适配**：官方 API 与模拟操作可灵活切换，适配层隔离差异。
